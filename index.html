什么是用户模式，什么是内核模式及其区别：

  1. Kernel Mode
    在内核模式下，代码具有对硬件的所有控制权限。可以执行所有CPU指令，可以访问任意地址的内存。内核模式是为操作系统最底层，最可信的函数服务的。在内核模式下的任何异常都是灾难性的，将会导致整台机器停机。
  2. User Mode
    在用户模式下，代码没有对硬件的直接控制权限，也不能直接访问地址的内存。程序是通过调用系统接口(System APIs)来达到访问硬件和内存。在这种保护模式下，即时程序发生崩溃也是可以恢复的。在你的电脑上大部分程序都是在用户模式下运行的。

**内核模式： **代码具有访问硬件和内存的所有权限，位于操作系统最底层，属于可信计算函数服务的。

用户模式：代码没有访问硬件和内存的直接权限，只能通过调用系统结构来访问硬件和内存，大多数软件都是运行在用户模式下的，任何异常都可以恢复

病毒感染文件的步骤：

病毒如何感染文件：
在文件中添加一个新节，在新节中添加病毒代码，和返回宿主程序的代码；同时修改PE的入口点，使其指向新添加的病毒代码入口；程序执行时先执行病毒代码，然后跳转至宿主程序。
具体过程：

  1. 判断目标文件开头的两个字节是否是MZ
  2. 判断PE文件标记PE
  3. 判断是否是病毒标记，如果是的话，则跳出执行宿主代码，否则继续；
  4. 获得数据目录的个数
  5. 找到节表的起始偏移
  6. 得到节表的末尾偏移
  7. 开始写入节表
  8. 在新添加的节中添加病毒代码
  9. 把当前文件位置设为文件末尾

段页管理

段式管理是通过段表进行的，它包括段号或段名、段起点、装入位、段的长度等。此外还需要主存占用区域表、主存可用区域表。

页式管理是将各进程的虚拟空间划分成若干个长度相等的页，然后把页式虚拟地址与内存地址建立一一对应页表，并用相应的硬件地址变换机构，来解决离散地址变换问题。

网站挂马技术：

网站挂马技术就是在网页中插入一段恶意代码，当浏览者打开被挂马的网页的时候，恶意代码就会被执行；然后木马的服务端程序或种子就会下载到浏览者本地并运行，进而控制浏览者的本机。

常见的挂马技术：

  • js挂马
  • 图片伪挂马
  • 网络钓鱼挂马
  • 伪装挂马

虚实地址转换：

  1. 已知条件：

给定的虚拟地址： 0x01AF5518

CR3寄存器的值： 0xAA0E5000

[0xAA0E5018] = 0x3D955867

[0x3D955BD4] = 0x7095e187

步骤：

  1. 把给定的虚拟地址按照 高10位 中间10位 低12位拆分为三部分。

    先转换为2进制

    0000 0001 1010 1111 0101 0101 0001 1000

    高10位：0000000110 -> 0x06 页目录索引

    中10位：1011110101 -> 0x2F5 页表索引

    低12位：010100011000 ->0x518 偏移

  2. 根据当前的CR3寄存器中的物理地址定位页目录表基址

    页目录表基地址（PDE）= CR3 = 0xAA0E5000

  3. 计算页表项的地址

    页表项的地址粗放在页目录表的第6个位置：

    [0xAA0E5000+4*6]=[0xAA0E5018]

    页表基地址（PTE）= 0x3D955000

  4. 计算页面基地址

    [0x3D955000+0x2F5*4]=[0x3D955BD4]

    页面基地址为 0x7095E000

  5. 计算真实物理地址

    [0x7095E000+0x518]=0x7095E518

  1. 已知条件：

    虚拟地址： 018a0b3c

    CR3|PDB : 38200000

    [38200018]=36fcf867

    [38fcf280] = 38180867

  2. 0000 0001 1000 1010 0000 1011 0011 1100

    页目录索引：0x6

    页表索引： 0xA0

    偏移： 0xB3C

  3. 页目录基地址

    PDB =CR3= 38200000

  4. 页表基地址

    [38200000+0x6*4]=[38200018]=36fcf867

    页表基地址为=36fcf000

  5. 页面基地址

    [36fcf000+0xA0*4]=[36fcf280] = 38180867

    ​ 0x280

    页面基地址为：38180000

病毒如何感染文件

在文件中添加一个新的节，然后把病毒代码和病毒执行后返回宿主程序的代码写入新添加的节中；同时修改PE文件头中入口点，使其指向新添加的病毒代码入口。
程序执行时，先执行病毒代码，病毒代码执行后再跳转至执行宿主程序。

  1. 判断目标文件开头的两个字节是否是 MZ
  2. 判断PE文件标记PE
  3. 判断感染标记，如果一倍感染则跳出，继续执行宿主程序，否则继续；
  4. 获得数据目录的个数
  5. 得到节表的起始位置
  6. 得到节表的末尾偏移
  7. 开始写入节表
  8. 在新添加的节中添加病毒代码
  9. 把当先文件位置设为文件末尾

病毒如何感染文件：
在文件中添加一个新节，在新节中添加病毒代码，和返回宿主程序的代码；同时修改PE的入口点，使其指向新添加的病毒代码入口；程序执行时先执行病毒代码，然后跳转至宿主程序。
具体过程：

  1. 判断目标文件开头的两个字节是否是MZ
  2. 判断PE文件标记PE
  3. 判断是否是病毒标记，如果是的话，则跳出执行宿主代码，否则继续；
  4. 获得数据目录的个数
  5. 找到节表的起始偏移
  6. 得到节表的末尾偏移
  7. 开始写入节表
  8. 在新添加的节中添加病毒代码
  9. 把当前文件位置设为文件末尾
